## Authentication Methods

None Documentation supports multiple authentication methods to ensure secure access to your documentation. Choose the method that best fits your organization's security requirements and user management systems.

<Tabs>
  <Tab title="API Tokens" icon="key">
    Use API tokens for programmatic access to None Documentation's APIs. Tokens provide granular permissions and can be easily revoked if compromised.
    
    <Steps>
      <Step title="Generate Token" icon="plus">
        Navigate to your account settings and create a new API token.
      </Step>
      <Step title="Set Permissions" icon="shield">
        Choose read, write, or admin permissions for the token.
      </Step>
      <Step title="Use in Requests" icon="send">
        Include the token in your API requests using the Authorization header.
      </Step>
    </Steps>
  </Tab>
  <Tab title="OAuth" icon="lock">
    Integrate with popular identity providers like GitHub, Google, and Okta for seamless single sign-on.
    
    <Callout kind="info">
      OAuth integration requires administrator privileges to configure.
    </Callout>
  </Tab>
  <Tab title="SAML" icon="user">
    For enterprise customers, SAML authentication ensures compliance with organizational security policies.
  </Tab>
</Tabs>

## API Authentication

When making API calls to None Documentation, include your authentication credentials in the request headers.

<CodeGroup tabs="cURL,JavaScript">
  ```bash
curl -H "Authorization: Bearer YOUR_API_TOKEN" https://api.none-docs.com/v1/repos
  ```
  ```javascript
const headers = {
  'Authorization': `Bearer ${process.env.NONE_DOCS_TOKEN}`,
  'Content-Type': 'application/json'
};

const response = await fetch('https://api.none-docs.com/v1/repos', { headers });
  ```
</CodeGroup>

<Callout kind="alert">
  Never commit API tokens to version control. Use environment variables instead.
</Callout>

## Troubleshooting Authentication Issues

Common authentication problems and their solutions.

<ExpandableGroup>
  <Expandable title="Token Expired" default-open="false">
    API tokens expire after 30 days by default. Generate a new token in your account settings.
  </Expandable>
  <Expandable title="Invalid Permissions" default-open="false">
    Check that your token has the required permissions for the API endpoint you're accessing.
  </Expandable>
  <Expandable title="OAuth Configuration" default-open="false">
    Ensure your OAuth application is properly configured with the correct redirect URIs.
  </Expandable>
</ExpandableGroup>

| Error Code | Description | Solution |
|------------|-------------|----------|
| 401 | Unauthorized | Check your API token |
| 403 | Forbidden | Verify permissions |
| 429 | Rate limited | Wait before retrying |
